FEAT-01：カメラ数値制御・焦点距離・正射影切り替え
概要

JavaScript側 実装仕様
Perspective / Orthographic 切り替え
シーン初期化時に使用するカメラクラスを projection パラメータで分岐させること。
// projection === 'orthographic' の場合
camera = new THREE.OrthographicCamera(
    -ortho_scale, ortho_scale,
    ortho_scale / aspect, -ortho_scale / aspect,
    0.01, 1000
);
// projection === 'perspective' の場合（既存処理）
const fov = 2 * Math.atan(24 / (2 * focal_length)) * (180 / Math.PI);
camera = new THREE.PerspectiveCamera(fov, aspect, 0.01, 1000);

注意: 既存の OrbitControls はカメラ種別を問わず動作するが、切り替え時には controls.object を新しいカメラに再バインドすること。
数値入力UIの同期
既存のキーフレームUIとスライダーUIの数値は常に双方向同期すること。数値入力フィールドを変更した場合にビューポートが即時更新されること。

JavaScript側 実装
// シーン初期化時
const dirLight = new THREE.DirectionalLight(0xffffff, 1.2);
dirLight.position.set(1.0, 2.0, 1.0);
scene.add(dirLight);

const ambLight = new THREE.AmbientLight(0x404040, 0.4);
scene.add(ambLight);

const hemiLight = new THREE.HemisphereLight(0x87CEEB, 0x8B4513, 0.3);
scene.add(hemiLight);

// パラメータ更新時
function updateLighting(params) {
    dirLight.position.set(params.dir_x, params.dir_y, params.dir_z);
    dirLight.color.set(params.dir_color);
    dirLight.intensity = params.dir_intensity;
    ambLight.color.set(params.amb_color);
    ambLight.intensity = params.amb_intensity;
    hemiLight.color.set(params.hemi_sky);
    hemiLight.groundColor.set(params.hemi_ground);
    hemiLight.intensity = params.hemi_intensity;
}

推奨デフォルト値: Normalマップ生成に適したデフォルト値は dir_intensity=1.2、amb_intensity=0.4 程度。AmbientLight が強すぎると法線の陰影情報が失われるため注意。

FEAT-04：Rokoko リターゲット JSON 中継カスタムノード
概要
Rokoko Studio でエクスポートしたリターゲットマップ JSON を読み込み、ユーザーの自前アニメーションモデル（FBX/GLB）のボーン名を Yedp_Rig.glb のボーン名に変換した上で Yedp Action Director に渡す新規カスタムノードを実装する。
このノードは Yedp Action Director の既存アニメーション読み込みパイプラインへの「前段アダプター」として機能する。
実装前の必須確認作業
実装開始前に以下を必ず実行し、Yedp_Rig.glb の実際のボーン名一覧を取得すること。このリストが FEAT-04 の中核データとなる。
import trimesh
scene = trimesh.load('path/to/Yedp_Rig.glb')
for name, mesh in scene.geometry.items():
    print(name)
# または
import pygltflib
glb = pygltflib.GLTF2().load('Yedp_Rig.glb')
for node in glb.nodes:
    print(node.name)
Rokoko リターゲット JSON フォーマット
Rokoko Studio からエクスポートされるリターゲットマップのフォーマットは以下を想定する。実際のフォーマットが異なる場合は以下に準じて柔軟にパースすること。
{
  "retarget": [
    { "source": "mixamorig:Hips",    "target": "Hips" },
    { "source": "mixamorig:Spine",   "target": "Spine" },
    { "source": "mixamorig:LeftArm", "target": "LeftArm" },
    ...
  ]
}

フォーマット確認: 実際に Rokoko Studio からエクスポートしたサンプル JSON を取得し、フォーマットが上記と異なる場合は parser を調整すること。target 側の値が Yedp_Rig.glb のボーン名と一致しているかを確認する。

処理フロー
1. retarget_json_path を読み込み、source→target の辞書を構築する
2. animation_path のFBX/GLBを trimesh または pygltflib で読み込み、ボーン名一覧を取得する
3. source ボーン名を target（Yedpボーン名）に変換した新規GLBを一時ファイルに書き出す
4. 書き出したGLBのパスを STRING として返す
5. Yedp Action Director ノードの animation_path 入力に接続することで、通常通り動作する
実装コード骨格
import json, os, trimesh
from pathlib import Path

class YedpRokokoRetargeter:
    @classmethod
    def INPUT_TYPES(cls):
        return {'required': {
            'animation_path':    ('STRING', {'default': ''}),
            'retarget_json_path':('STRING', {'default': ''}),
            'frame_start':       ('INT',    {'default': 0, 'min': 0}),
            'frame_end':         ('INT',    {'default': 30, 'min': 1}),
            'frame_step':        ('INT',    {'default': 1, 'min': 1}),
        }}
    RETURN_TYPES = ('STRING',)
    RETURN_NAMES = ('retargeted_animation_path',)
    FUNCTION = 'retarget'
    CATEGORY = 'Yedp/Animation'

    def retarget(self, animation_path, retarget_json_path,
                 frame_start, frame_end, frame_step):
        # 1. リターゲットマップ読み込み
        with open(retarget_json_path, 'r', encoding='utf-8') as f:
            retarget_data = json.load(f)
        bone_map = {r['source']: r['target']
                    for r in retarget_data.get('retarget', [])}

        # 2. アニメーション読み込み・ボーン名変換
        scene = trimesh.load(animation_path)
        renamed = rename_bones(scene, bone_map)  # 実装必要

        # 3. 一時GLBとして書き出し
        out_dir = os.path.join('ComfyUI', 'input', 'yedp_anims', 'retargeted')
        os.makedirs(out_dir, exist_ok=True)
        out_path = os.path.join(out_dir, f'retargeted_{Path(animation_path).stem}.glb')
        renamed.export(out_path)
        return (out_path,)
エラーハンドリング要件
retarget_json_path が存在しない場合: FileNotFoundError を ComfyUI エラーとして表示
animation_path のボーン名がリターゲットマップに存在しない場合: 警告ログを出力し、元のボーン名のまま処理を継続（中断しない）
出力GLBの書き出しに失敗した場合: 詳細エラーメッセージを表示
